# Часть 1: Создание проекта и анализ pom.xml

1. Что находится в секции `<parent>` и зачем она нужна?

Ответ: В секции <parent> указан spring-boot-starter-parent. Это специальный родительский POM, который управляет версиями всех Spring Boot зависимостей, плагинов и настраивает свойства проекта. Он нужен для упрощения конфигурации: наследуя его, проект автоматически получает согласованные версии библиотек и стандартную конфигурацию сборки.

2. Почему у большинства зависимостей нет версий?

Ответ: Потому что версии унаследованы от spring-boot-starter-parent. В родительском POM определён Bill of Materials (BOM), который содержит согласованный набор версий для всех Spring Boot стартеров. Это гарантирует совместимость библиотек между собой и избавляет разработчика от необходимости указывать версии вручную.

3. Что делает `spring-boot-maven-plugin`?

Ответ: Плагин собирает проект в исполняемый JAR-файл с встроенным сервером приложений (fat JAR).

4. Какие транзитивные зависимости подключает `spring-boot-starter-web`? Выполните `mvn dependency:tree` и опишите 3–5 самых важных.

Ответ:
При выполнении команды mvn dependency:tree можно увидеть, что spring-boot-starter-web подключает:
- spring-boot-starter-tomcat — встроенный сервер Tomcat
- spring-webmvc — Spring MVC для создания REST контроллеров
- spring-web — основные HTTP-абстракции
- jackson-databind — для сериализации/десериализации JSON
- spring-boot-starter-json — поддержка JSON формата
- 
  транзитивные зависимости
1. spring-boot-starter-tomcat — встроенный сервер Tomcat (включает tomcat-embed-core и tomcat-embed-websocket), который позволяет запускать веб-приложение без внешнего сервера.
2. spring-webmvc — модуль Spring MVC, содержащий контроллеры, модели и представления для построения REST API и веб-приложений.
3. spring-web — базовый модуль Spring для работы с HTTP, включающий клиент RestClient и основные веб-утилиты.
